    **CameraManager** is an `@Observable @MainActor` class (NOT a singleton — one instance per camera session). It encapsulates all AVFoundation logic.

    ```swift
    import AVFoundation
    import UIKit
    import Observation

    @Observable
    @MainActor
    final class CameraManager: NSObject {
        // MARK: - Published State
        var isMultiCamSupported: Bool = false
        var permissionGranted: Bool = false
        var capturedImage: UIImage?  // Set after capture + compositing
        var rearPreviewLayer: AVCaptureVideoPreviewLayer?
        var frontPreviewLayer: AVCaptureVideoPreviewLayer?

        // MARK: - Private
        private var multiCamSession: AVCaptureMultiCamSession?
        private var singleSession: AVCaptureSession?
        private var rearPhotoOutput: AVCapturePhotoOutput?
        private var frontPhotoOutput: AVCapturePhotoOutput?
        private var singlePhotoOutput: AVCapturePhotoOutput?

        private var rearImage: UIImage?
        private var frontImage: UIImage?
        private var captureCompletion: (() -> Void)?

        // MARK: - Session Accessors
        var activeSession: AVCaptureSession? {
            multiCamSession ?? singleSession
        }
    }
    ```

    **Methods to implement:**

    1. `func checkPermission()` — Check `AVCaptureDevice.authorizationStatus(for: .video)`. If `.notDetermined`, call `requestAccess`. Set `permissionGranted` accordingly.

    2. `func setupSession()` — Called after permission granted:
       - Check `AVCaptureMultiCamSession.isMultiCamSupported`. Set `isMultiCamSupported`.
       - If supported: create `AVCaptureMultiCamSession`, add rear wide-angle camera input + output, add front camera input + output. Create two `AVCaptureVideoPreviewLayer` instances.
       - If NOT supported: create standard `AVCaptureSession`, add rear camera only with single photo output. Create one preview layer.
       - Start session on a background thread: `DispatchQueue.global(qos: .userInitiated).async { session.startRunning() }`

    3. `func capturePhoto()` — Trigger capture:
       - If multi-cam: capture from both `rearPhotoOutput` and `frontPhotoOutput` using `AVCapturePhotoOutput.capturePhoto(with:delegate:)`. Use `AVCapturePhotoSettings()` for each.
       - If single-cam: capture from `singlePhotoOutput` only.
       - Set `self` as delegate (conform to `AVCapturePhotoCaptureDelegate`).

    4. `func compositeImages(rear: UIImage, front: UIImage) -> UIImage` — Draw rear image at full frame, draw front image at bottom-left corner:
       - Front inset size: `25%` of rear image width (per CONTEXT.md Decision 1)
       - Position: bottom-left, 16pt inset from edges
       - Corner radius on front inset: 12pt
       - 2pt white border stroke around front inset
       - Use `UIGraphicsImageRenderer` with rear image size
       - Draw rear image filling the entire canvas
       - Draw front image in a rounded rect at bottom-left with clipping path + stroke

    5. `func stopSession()` — Stop running session on background queue.

    **AVCapturePhotoCaptureDelegate** conformance:
    - `photoOutput(_:didFinishProcessingPhoto:error:)` — Convert `AVCapturePhoto` to UIImage via `photo.fileDataRepresentation()` then `UIImage(data:)`.
    - For multi-cam: track which output fired (rear vs front) by comparing the `output` parameter to stored references. When both images collected, call `compositeImages` and set `capturedImage`.
    - For single-cam: set `capturedImage` directly from the rear photo.

    **Important details:**
    - The class conforms to `NSObject` (required by `AVCapturePhotoCaptureDelegate`)
    - Camera device discovery: `AVCaptureDevice.DiscoverySession(deviceTypes: [.builtInWideAngleCamera], mediaType: .video, position: .back/.front)`
    - Photo settings: `AVCapturePhotoSettings()` with default JPEG output
    - All UI state updates must happen on MainActor (the class is already @MainActor)
    - Session configuration changes must be wrapped in `session.beginConfiguration()` / `session.commitConfiguration()`

    **1. Create `Circles/Moment/MomentCameraView.swift`:**

    Presented as `.fullScreenCover` from CircleDetailView (wired in Plan 03).

    ```swift
    import SwiftUI
    import AVFoundation

    struct MomentCameraView: View {
        let circleId: UUID
        let onCapture: (UIImage) -> Void   // Called with composited image
        @Environment(\.dismiss) private var dismiss
        @State private var cameraManager = CameraManager()
        @State private var flashOpacity: Double = 0

        var body: some View {
            ZStack {
                // Case 1: Permission denied
                if !cameraManager.permissionGranted {
                    permissionDeniedView
                } else {
                    // Camera viewfinder
                    cameraViewfinderView
                }
            }
            .ignoresSafeArea()
            .task {
                cameraManager.checkPermission()
                // Small delay to allow permission alert
                try? await Task.sleep(for: .milliseconds(500))
                if cameraManager.permissionGranted {
                    cameraManager.setupSession()
                }
            }
            .onChange(of: cameraManager.permissionGranted) { _, granted in
                if granted {
                    cameraManager.setupSession()
                }
            }
            .onChange(of: cameraManager.capturedImage) { _, image in
                if let image {
                    onCapture(image)
                }
            }
            .onDisappear {
                cameraManager.stopSession()
            }
        }
    }
    ```
