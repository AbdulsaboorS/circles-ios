# Phase 06.2: Core Screens Redesign — Context

**Gathered:** 2026-03-24
**Status:** Ready for planning

<domain>
## Phase Boundary

Full visual redesign of the four core screens — HomeView, CommunityView (My Circles + Public Explore), CircleDetailView, and FeedView — using the Phase 06.1 design system. Adds `is_public` flag to the circles schema (Supabase migration). No new features or business logic.

All screens get: AppBackground animated blobs (ZStack), AppCard surfaces, Color.appBackground / Color.cardSurface / Color.accent tokens, New York serif for greeting/hero headers, SF Pro for body.

</domain>

<decisions>
## Implementation Decisions

### Design reference
- **D-01:** Primary reference: user-provided mobile dark dashboard screenshot (screenshot 1). Layout, component hierarchy, and section structure should match this reference, with updated colors (#0E0B08 bg, #E8834B accent instead of navy).
- **D-02:** Explore reference: user-provided web bubble layout screenshot (screenshot 2). Adapted to mobile as staggered 2-column bubble grid.

---

### HomeView

- **D-03:** Layout: Dashboard card layout. Greeting header at top → streak card → "Daily Intentions" section → habit cards → "Community Circles" section. ScrollView-based (not List).
- **D-04:** Greeting header: New York serif, large, time-aware ("Peaceful Midnight, Abdul Saboor 🌙" / "Good Morning ☀️" / "Good Afternoon ☀️" / "Good Evening 🌙"). Uses user's first name from auth profile.
- **D-05:** Streak card: Clean AppCard. Shows streak number prominently, Islamic motivational quote (static or rotating), longest streak. No "Share Light" button (backlog item). No completion progress indicator.
- **D-06:** Habit cards: Each habit is its own AppCard row. Left side: Islamic illustration image (user sources assets — use a placeholder SF Symbol until assets arrive). Habit name. A tappable chip. Tap habit card → HabitDetailView (Phase 06.3).
- **D-07:** "Daily Intentions" uses SectionHeader component from 06.1.
- **D-08:** "Community Circles" section at bottom of HomeView: SectionHeader + a short preview list of public circles (2-3 cards) with circle name, member count, avatar stack, "Join Circle" ChipButton. Tapping a circle card navigates to CommunityView Explore tab. This is a discovery surface — same data as Public Explore.

  **ACCEPTED DEVIATION (Phase 06.2):** Community Circles cards on HomeView navigate to CircleDetailView directly, NOT to CommunityView Explore tab. Reason: switching tabs across unrelated NavigationStacks requires a coordinator/shared state mechanism that is out of scope for this redesign phase. The navigation destination is functionally equivalent for the user (they see the circle). Tab-switch navigation may be revisited in a future polish pass.

- **D-09:** AppBackground blobs behind the entire scrollable content via ZStack.

---

### CommunityView

- **D-10:** Segmented control at the top: "My Circles" | "Explore". iOS-native `Picker` with `.segmented` style.
- **D-11:** My Circles tab: AppCard list of joined circles. Each card shows circle name, member count, X/Y checked in today. Tap → CircleDetailView.
- **D-12:** Public Explore tab: Staggered 2-column bubble grid. Each circle is a circular/rounded card with circle name + member count. Tappable. Circle cards use `.ultraThinMaterial` dark / white+shadow light (AppCard style but circular shape).
- **D-13:** `is_public` flag added to circles Supabase table via migration. Public Explore query filters `is_public = true`.
- **D-14:** Add Circle button (create/join) stays in toolbar — same as existing.

---

### CircleDetailView

- **D-15:** Feed-first layout. Members shown as a horizontal scroll of avatar chips at the top (compact). Active Moment window banner sits directly below the members strip (when a window is open). Feed fills the rest of the screen.
- **D-16:** Members strip: horizontal ScrollView of avatar + name chips. Checked-in members get a ✓ indicator. Keeps existing data, just reskins.
- **D-17:** Active window banner: amber accent card (Color.accent bg, white text) showing countdown timer when window is open. Disappears when window is closed.
- **D-18:** Feed: FeedView embedded / reused here. Moment cards, check-in cards, streak milestones all use new card designs.
- **D-19:** AppBackground behind the whole screen. Navigation title = circle name.

---

### FeedView + Feed Cards

- **D-20:** Moment cards (photo posts): Full-width photo fills the AppCard. Caption + user name overlaid at the bottom with a dark gradient. ⭐ on-time badge in the top-right corner if posted within window. Reactions bar below the card (outside the photo, in the card padding).
- **D-21:** Habit check-in cards: AppCard with habit name, user name, check-in time, streak count. Compact horizontal layout.
- **D-22:** Streak milestone cards: AppCard with 🔥 emoji, milestone text ("Abdul Saboor hit a 7-day streak!"), reaction bar.
- **D-23:** All cards use AppCard (ultraThinMaterial dark / white+shadow light, 16pt radius).

---

### Schema change

- **D-24:** Add `is_public BOOLEAN DEFAULT false` to `circles` / `halaqas` table via Supabase migration. Plan must include the SQL migration step. Existing circles default to private.

---

### Claude's Discretion

- Rotating vs static Islamic quote on streak card
- Exact bubble grid sizing and spacing in Explore
- Avatar chip exact layout in members strip
- Animation on the segmented control tab switch

</decisions>

## Specific Ideas
