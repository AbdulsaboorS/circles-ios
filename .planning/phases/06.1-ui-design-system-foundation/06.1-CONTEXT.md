# Phase 06.1: UI Design System Foundation - Context

**Gathered:** 2026-03-24
**Status:** Ready for planning

<domain>
## Phase Boundary

Establish the complete design system that all subsequent redesign phases (06.2, 06.3) build on top of. This includes: color tokens (light + dark mode), typography scale, reusable SwiftUI components (cards, buttons, chips, background blobs), background blob animation system, sunrise/sunset auto dark mode logic, and app icon design.

No screen-level redesign happens in this phase — only the foundation layer. Screen rebuilds begin in Phase 06.2.

</domain>

<decisions>
## Implementation Decisions

### Dark mode background color
- **D-01:** Warm near-black `#0E0B08` — a brownish-black with warm undertone, not navy. Clearly differentiates from Pillars app while staying premium and dark.

### Light mode background color
- **D-02:** Warm cream background (~`#F5F0E8` range) with warm beige blob shapes. White cards on cream base.

### Accent color
- **D-03:** Amber `#E8834B` — kept from existing codebase. Used for CTAs, active states, streak highlights, and the tab bar tint.

### Typography
- **D-04:** New York serif for greeting headers and hero text (e.g., "Peaceful Morning, Abdul Saboor"). Built into iOS 17+, no bundling required.
- **D-05:** SF Pro for all body text, labels, captions, and UI elements. System default.

### Token architecture
- **D-06:** Single `DesignTokens.swift` file with static Color extension properties. No xcassets Color Sets (incompatible with programmatic sunrise/sunset switching).
- **D-07:** Semantic naming: `Color.appBackground`, `Color.cardSurface`, `Color.accent`, `Color.textPrimary`, `Color.textSecondary`, `Color.blobPrimary`, `Color.blobSecondary`, etc.
- **D-08:** The existing `Color+Hex.swift` hex extension is kept as a utility. DesignTokens.swift uses it internally.

### Background blob system
- **D-09:** Animated — blobs breathe/drift slowly (3–5 second cycle, gentle opacity and scale pulse). Meditative feel, matches the spiritual tone of the app.
- **D-10:** Implementation: SwiftUI Canvas or custom Shape + `.animation(.easeInOut(duration: 4).repeatForever(autoreverses: true))`. Two blobs per screen — one large primary, one smaller secondary at opposite corner.
- **D-11:** Dark mode blobs: warm near-black base + forest green blobs (~`#1A3A2A` range).
- **D-12:** Light mode blobs: cream base + warm beige blobs (~`#EDE0C8` range, slightly darker than background).
- **D-13:** Blob opacity: ~0.5–0.6, soft gaussian-style blur radius ~80–120pt so they read as ambient color, not hard shapes.

### Card / glassmorphism system
- **D-14:** Dark mode cards: `.ultraThinMaterial` SwiftUI material for glassmorphism effect. No custom blur needed — iOS built-in material looks correct.
- **D-15:** Light mode cards: white background + very subtle shadow (`Color.black.opacity(0.06)`, radius 8, y 2). Clean, not glassy.
- **D-16:** Shared card corner radius: 16pt.

### Reusable components to build
- **D-17:** `AppCard` — rounded card container with automatic light/dark surface. Takes any content view.
- **D-18:** `PrimaryButton` — filled amber CTA button with standard padding and corner radius.
- **D-19:** `ChipButton` — small pill-shaped chip (e.g., "Breathe & Reflect", "Join Circle"). Tappable or static variant.
- **D-20:** `AppBackground` — full-screen animated blob background. Wraps any screen via `.background(AppBackground())` or ZStack.
- **D-21:** `SectionHeader` — consistent section label style (e.g., "Daily Intentions", "Community Circles").

### Dark/light mode switching
- **D-22:** Auto-switches at user's sunrise (→ light) and sunset (→ dark) using Adhan library + stored `latitude`/`longitude` from Phase 6 user profile.
- **D-23:** Manual override toggle in Settings tab — user can lock to light, dark, or auto.
- **D-24:** `ThemeManager.swift` — @Observable @MainActor singleton. Exposes `colorScheme: ColorScheme`. Applied via `.preferredColorScheme(themeManager.colorScheme)` at root ContentView level.
- **D-25:** Schedule: compute next sunrise/sunset on app launch and when app returns to foreground. Use `Timer` to fire the switch. Recalculate daily.

### App icon
- **D-26:** Concept: geometric Islamic tessellated pattern using circular forms.
- **D-27:** Treatment: warm near-black background (`#0E0B08`) with amber/gold geometric pattern overlay.
- **D-28:** Claude generates the icon as a SwiftUI Canvas view → screenshot to PNG → export all required App Store sizes via Xcode. No external tools needed.

### Claude's Discretion
- Exact blob shape (ellipse vs freeform bezier)
- Specific animation easing curve and timing
- Exact shadow values for light mode cards
- DesignTokens.swift file organization (grouped by category or flat list)
- Exact typography size scale (title, headline, body, caption sizing values)

</decisions>

<specifics>
## Specific Ideas

- Reference designs: user provided 3 screenshots — mobile dashboard (dark mode), web explore page (dark mode bubble layout), web dashboard (light mode). Dark and light palettes extracted from these.
- "Not the Pillars navy" — warm near-black replaces the current `#0D1021` navy used throughout the existing codebase.
- The greeting header style: "Peaceful Morning, Abdul Saboor ☀️" — large serif text, context-aware time-of-day emoji, very personal feel.
- Blobs should feel "meditative" — slow, calming motion, not distracting.
- Islamic geometric icon: tessellated circular forms, amber/gold on dark background — signals Islamic identity immediately on the App Store.

</specifics>
