# Phase 06.3: Secondary Screens Redesign — Context

**Gathered:** 2026-03-25
**Status:** Ready for planning

<domain>
## Phase Boundary

Apply the Phase 06.1 design system to all screens not touched in Phase 06.2, plus feature-level upgrades to HabitDetailView and ProfileView. Screens in scope: HabitDetailView, ProfileView, MomentCameraView, MomentPreviewView, and all 5 onboarding screens (ProfileSetupView, HabitSelectionView, RamadanAmountView, AIStepDownView, LocationPickerView). Includes redesigned empty/loading/error states and a final polish pass with navigation transitions.

No new features outside the explicitly listed upgrades. Business logic (HabitService, FeedService, etc.) is not changed — only UI and a small number of data model additions for notes fields.

</domain>

<decisions>
## Implementation Decisions

---

### HabitDetailView

- **D-01:** Keep 28-day dot grid layout (SwiftUI.Circle() dots, 7-column LazyVGrid). Completed days = `Color.accent` fill, incomplete = `Color.textSecondary.opacity(0.2)`. Background: AppBackground. AppCard wraps the hero section and grid section.
- **D-02:** Per-day notes — notes live in HabitDetailView only (not on check-in). User taps a day dot → sheet opens with a TextField to add/edit a note for that day. Note saved to `habit_logs.notes` column (new column — requires Supabase migration: `ALTER TABLE habit_logs ADD COLUMN IF NOT EXISTS notes TEXT`). Notes appear below each dot as a small indicator dot (or line) when present.
- **D-03:** AI step-down plan section — shown below the calendar. Displays the habit's `acceptedAmount` goal and AI-generated milestone plan from onboarding. Read-only display + two edit modes:
  1. **Manual refinement:** User can tap to edit the `acceptedAmount` text inline (saved to `habits.accepted_amount`). A separate `plan_notes` free-form text field lets the user annotate the plan (saved to `habits.plan_notes` — new column: `ALTER TABLE habits ADD COLUMN IF NOT EXISTS plan_notes TEXT`).
  2. **AI-assisted re-generation:** "Refine with AI" button calls `GeminiService` with current habit + accepted amount + plan notes as context. Returns a new step-down plan. User confirms before saving.
- **D-04:** Typography: habit name in `Font.appTitle` (New York serif), section labels via `SectionHeader` component, stats in `Font.appHeadline`. All colors via design tokens.

---

### ProfileView

- **D-05:** Reference: user-provided mockup (light mode, cream bg). Matches design system light mode tokens.
- **D-06:** Layout (top → bottom): Navigation title "Profile" (inline) + settings gear toolbar button → circular avatar → name + "Member since [Month Year]" → stats AppCard → "My Habits" section → "My Circles" section.
- **D-07:** Avatar: `SwiftUI.Circle()` with `Color.accent.opacity(0.15)` fill + `person.fill` SF Symbol tinted `Color.accent`. No photo upload in this phase.
- **D-08:** "Member since" date: derived from `auth.session?.user.createdAt` — formatted as "Member since January 2025".
- **D-09:** Stats AppCard (3 columns): Total Days (count of all completed `habit_logs` rows for user) / Best Streak (`streak.longestStreak`) / Circles (count of user's circles). Icons: `flame.fill` amber for Days + Streak, `person.2.fill` for Circles.
- **D-10:** "My Habits" section: `SectionHeader` + 2-column chip grid. Each chip shows colored dot (fixed palette cycling per habit) + habit name + 🔥 current streak number. Chip style matches `ChipButton` outlined variant.
- **D-11:** "My Circles" section: `SectionHeader` + `AppCard` list rows. Each row: `person.2.fill` icon (amber-tinted circle bg) + circle name (bold) + member count (secondary) + chevron. Tap → `CircleDetailView`.
- **D-12:** Settings gear → `SettingsView` sheet. Contains: Theme mode picker (Auto / Light / Dark — maps to `ThemeManager.ThemeMode`), notification preferences toggle, sign out button. Debug tools remain in `#if DEBUG` block.

---

### MomentCameraView

- **D-13:** Keep fully dark/native — camera preview fills screen, blob background is irrelevant. Only changes: replace any hardcoded `Color(hex: "0D1021")` navy with `Color.appBackground`, apply `Color.accent` to the shutter button fill/tint, `Color.textPrimary` for labels and close button. Permission-denied state uses design tokens.

---

### MomentPreviewView

- **D-14:** Full design system. Apply `AppBackground` blobs behind the screen (ZStack). Caption input wrapped in an `AppCard` (glassmorphism). `PrimaryButton` for "Post Moment" CTA. "Retake" as a `ChipButton`. Replace `Color(hex: "0D1021")` and `Color(hex: "1A1D35")` with design tokens throughout.

---

### Onboarding Screens (full visual redesign — all 5)

- **D-15:** All 5 screens get `AppBackground` behind content. `PrimaryButton` for all primary CTAs. Replace hardcoded `Color(hex: "0D1021")` with `Color.appBackground`.
- **D-16:** Step progress indicator: small pill/dot row at the top of each screen showing current step out of 5. Active dot = `Color.accent`, inactive = `Color.textSecondary.opacity(0.3)`.
- **D-17:** Welcome/hero text upgraded to New York serif (`Font.appHeroTitle` or `Font.appTitle`) for the main heading on each screen. Subheadings in `Font.appSubheadline`.
- **D-18:** Islamic step headers: each onboarding screen gets a contextual SF Symbol illustration (large, `Color.accent` tinted) as a visual anchor above the heading:
  - ProfileSetupView: `person.fill` or `person.crop.circle`
  - HabitSelectionView: `star.fill` or `moon.fill`
  - RamadanAmountView: `moon.stars.fill`
  - AIStepDownView: `sparkles` or `brain`
  - LocationPickerView: `location.fill`
- **D-19:** `HabitSelectionView` habit tiles upgraded to AppCard-style rounded tiles (currently using raw `.background` inline). `ChipButton` or AppCard tile for each habit.
- **D-20:** `TextField` styling: replace `Color(hex: "1A1D35")` and `Color.white.opacity(0.07)` inline styles with `Color.lightBackground.opacity(0.08)` in dark / `Color.darkBackground.opacity(0.06)` in light. Consistent across all onboarding TextFields.

---

### Empty / Loading / Error States

- **D-21:** Loading state: `ProgressView().tint(Color.accent)` centered, no skeleton screens.
- **D-22:** Empty states: SF Symbol illustration (large, `Color.accent` opacity tinted) + Islamic dua/encouragement copy in `Font.appSubheadline` + `Color.textSecondary`. Examples:
  - No circles: `person.2.fill` → "Invite your brothers and sisters. Start your first circle."
  - No feed items: `heart.fill` → "No moments yet. Be the first to post today."
  - No habits checked in: `moon.stars.fill` → "Begin today — every journey starts with bismillah."
  - No habits in HabitDetailView history: `checkmark.circle` → "No check-ins yet. Your history will appear here."
- **D-23:** Error states: `AppCard` with `Color.red.opacity(0.15)` background, `xmark.circle.fill` SF Symbol tinted red, error message in `Font.appCaption` + retry `ChipButton`.

---

### Polish Pass + Navigation Transitions

- **D-24:** Add navigation transitions using SwiftUI `.navigationTransition()` (iOS 18+) for push/pop. Tab switches get a subtle `.opacity` or default crossfade. Use `.matchedTransitionSource` / `.navigationTransition(.zoom(...))` for drill-downs (e.g., habit card → HabitDetailView) if supported.
- **D-25:** Final check pass: after all screens redesigned, run through every screen in Simulator. Fix any remaining `Color(hex: ...)` hardcodes, missing empty states, or token inconsistencies found during the pass.

---

### Claude's Discretion

- Exact colored dot palette for habit chips in ProfileView (fixed cycling vs hash-based color assignment)
- Exact step indicator component style (dots vs pill-progress)
- Navigation transition type per screen pair (zoom vs push vs default)
- GeminiService prompt wording for plan re-generation
- Exact layout proportions and padding within ProfileView sections

</decisions>

<specifics>
## Specific Ideas

- ProfileView reference mockup provided by user (2026-03-25): light mode, cream bg, serif name, 3-column stats AppCard, 2-column habit chips with colored dots + flame streak, My Circles list with group icon rows.
- "Member since January 2025" from `auth.session?.user.createdAt` — not from `profiles` table.
- Notes indicator on dot: small amber dot below the day number when a note exists for that day.
- AI plan re-generation prompt should include: habit name + current `acceptedAmount` + user's `plan_notes` + current streak for context.
- Step progress indicator should be driven by `OnboardingCoordinator.currentStep` — no separate state needed.

</specifics>
