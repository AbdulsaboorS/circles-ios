**Gathered:** 2026-03-31
**Status:** Ready for planning

<domain>
## Phase Boundary

Finalize both onboarding flows (Amir/Circle-creator and Joiner/Member) with:
- Full narrative step sequences as designed
- Deep link detection and routing
- Data persistence before auth (UserDefaults cache)
- Push notification permission (Islamic framing)
- SwiftUI animated previews (placeholder for real video later)
- Auth-last architecture (onboarding completes → auth gate as final step)
- Lottie animations for AI generation loader

<decisions>
## Implementation Decisions

### Architecture — Auth-Last
- Current flow: auth → onboarding. New flow: full onboarding → auth gate at the very end.
- All onboarding state held in memory + UserDefaults before auth succeeds.
- After auth: flush pending state (circle creation, habits, plans) to Supabase in one batch.
- ContentView routing needs a full rewrite to support unauthenticated onboarding traversal.

### Amir Flow — 7 Steps
1. **Landing Sanctuary** — SwiftUI animated preview (looping) + emotive copy + two CTAs: "Build My Circle" (primary) + "Join with Invite Code" (secondary)
2. **The Struggle** — Core habit selection, max 3, curated list (same as current Step 2)
3. **Circle Identity** — Circle name + gender setting. Mixed note: "Mixed circles are for families only. Allah is Witness to our intentions."
4. **Personal Intentions** — Private habit selection, max 2 (existing AmiirStep3PersonalView, update cap from 3→2). Visual: lock icon, slightly different color from shared steps. Copy: "These will never be seen by your circle."
5. **AI Generation** — Animated progress screen. Copy: "Generating your 28-day roadmaps based on the Sunnah..." + "You'll see them on your dashboard when ready." Plans fire in background (Task { } — fire and forget). No blocking.
6. **Foundation** — Preferred name + location (city for prayer times). Push notification permission ask during this step. Framing: "Enable the Adhan for your circle" — explain it powers the Moment notification window.
7. **Activation (Auth Gate)** — Copy: "Your sanctuary is ready. Save your progress and invite your first 2 brothers/sisters." Show preview of Evolving Heart or AI roadmap card to motivate save. Google + Apple Sign-In. Soul Gate (share) is DEFERRED — user lands on dashboard and gets a nudge to share from there.

### Joiner Flow — 7 Steps
1. **Landing Page** — Top 80% screen: SwiftUI animated preview (video placeholder). Bottom 20%: "Join the Circle I was invited to" CTA + manual "Enter Invite Code" input (always visible as fallback for deep link failures).
2. **Rich Circle Preview** — Circle name, group streak flame, member face pile (up to 5). Copy: "You've been invited. Join [name] and [N] others in '[Circle Name]'?" Deep link auto-fills the circle code; manual entry also supported.
3. **Circle Alignment** — "The group is tracking [Habit A] and [Habit B]. Which will you do with them?" Must pick ≥ 1 core habit to proceed. Transition (1.5s): "Iron sharpens iron. We were never meant to grow alone."
4. **Personal Habits** — Pick private habits, max 2. Visual: lock icon, private color treatment. Transition (1.5s): "Some growth is between you and your Creator."
5. **AI Generation** — Same as Amir Step 5. Background generation, reassurance copy.
6. **Identity** — Name + location. Push notification ask here too. Same framing: "Enable the Adhan for your circle."
7. **Auth Gate** — "Don't lose your spot in the circle. Save your progress to activate your roadmap." Show roadmap preview card. Google + Apple Sign-In.

### Deep Links — URL Scheme (Simple Now)
- Register URL scheme `circles://` in Xcode → Info.plist → URL Types.
- `CirclesApp.swift` already has `.onOpenURL { }` stub — extend it to extract circle code from `circles://join/ABC123`.
- If deep link detected: prefill the code on Joiner Landing (Step 1) and auto-submit into Rich Circle Preview (Step 2).
- Deep link fail case: always show "Enter Invite Code" button on Joiner Landing (Step 1) as fallback.
- Future upgrade: when web landing page ships (Phase 13), migrate to Universal Links (`https://joinlegacy.app/join/ABC123`). URL scheme stays as fallback.

### Data Persistence — UserDefaults
- Use UserDefaults to cache pending onboarding state before auth.
- Cache struct (Codable): circle name, gender setting, selected core habits, selected personal habits, generated plan data (if available), name, location, city.
- Save after AI Generation step (Step 5) — by then all selections are made.
- On app relaunch: check for pending state → if found, resume from where user left off.
- After auth succeeds: flush to Supabase, clear UserDefaults cache.
- Failure path: if auth fails after cache is set, don't wipe cache — let user retry.

### Push Notifications — Soft Ask
- Ask during Foundation/Identity step (where location is set).
- Framing: "Enable the Adhan for your circle" — explains it powers the Moment notification window for prayer times.
- If granted → register for remote notifications.
- If denied → show inline banner (NOT a blocking screen): "Without the Adhan notification, your Circle Moment window won't alert you. You can enable it anytime in Settings →"
- Include a `Settings.app` deeplink (`UIApplication.openSettingsURLString`).
- Never block onboarding for push denial.

### Soul Gate — Deferred to Dashboard
- Remove Soul Gate hard lock from Amir onboarding Step 4 (current AmiirStep4SoulGateView).
- After auth and landing on dashboard: show a persistent nudge card/banner: "Invite your first 2 brothers/sisters to activate your group streak."
- Soul Gate logic (ShareSheet) moves to HomeView or CircleDetailView post-auth.

### Animations & Visuals
- **Landing Sanctuary / Joiner Landing video:** SwiftUI animated previews as placeholders (NOT real video yet). Simulate app usage: animated check-in button glow, habit streak tick, group flame. Placeholder architecture: `AVPlayer` with local `.mp4` asset when real footage is ready — just swap the file.
- **Step transitions:** 1.5–2 second animated transitions between steps. Skippable (tap to advance).
- **AI Generation screen:** Lottie animation (spinning crescent or progress arc). Add `lottie-ios` via SPM.
- **Evolving Heart:** Custom SwiftUI `Shape` animation — heart that grows/fills over time. Used on AI Gen step and Auth Gate preview.
- **Auth Gate preview:** Scaled-down version of HabitDetailView roadmap card, animated in with `.transition(.move(edge: .bottom).combined(with: .opacity))`.

### Personal Intentions Step (Already Built)
- `AmiirStep3PersonalView.swift` exists from Phase 11.2 QA fixes.
- Changes needed: cap max from 3 → 2, add lock icon visual treatment, update copy to match new flow.
- No new DB work — `is_accountable = false` is the existing "personal/private" flag.
- Joiner flow needs equivalent view created (similar to Amir's, but filtered to non-circle habits).

### Lottie SPM
- Add `lottie-ios` package to project (GitHub: airbnb/lottie-ios, latest stable).
- Use for: AI generation loader, Evolving Heart fallback if SwiftUI animation is too complex.
- Source free animations from LottieFiles.com (Islamic/minimal theme).

### StepIndicator
- Update to 7 steps for both Amir and Joiner flows (currently 5 for Amir, 2 for Joiner).
- Animated pill progress — existing component, just update step count.

### UI Visual Language
- Shared steps: Midnight Green (`#1A2E1E`) background — existing MS design system.
- Private/Personal steps: slightly lighter green or add lock icon (`lock.fill` SF Symbol) in gold to visually flag "this is private."
- Push notification ask: Adhan bell icon (`bell.fill`) in gold alongside location step.
- Auth gate: warm overlay, show roadmap card preview in bottom half.

### Transition Copy (1.5s, skippable)
- Between Circle Identity → Personal Intentions: "Iron sharpens iron. We were never meant to grow alone." (1.5s)
- Between Personal Intentions → AI Generation: "Some growth is shared; some is between you and your Creator." (1.5s)
- Joiner between Circle Alignment → Personal Habits: "Shared growth is powerful, but some habits are just for you and Allah." (1.5s)
- All transitions tap-to-skip.

### Claude's Discretion
- Exact SwiftUI animation implementation details (PhaseAnimator vs withAnimation vs custom).
- Lottie file selection and customization.
- UserDefaults key naming and struct field naming.
- Exact layout/spacing of auth gate preview card.
- Order of Supabase batch writes after auth.

</decisions>
