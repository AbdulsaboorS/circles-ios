# Phase 11.4 — Circle Moment (BeReal Mechanic)
# CONTEXT — Implementation Decisions

**Goal:** Finalize the Circle Moment feature end-to-end — from prayer-time notification trigger through dual-camera capture, posting, and feed display. This is the core social mechanic of Circles.

---

## Locked Product Decisions (from prior planning)

### Trigger
- 30-minute window, randomized across the day's prayers (not always the same prayer)
- Push notification fires when the window opens
- Window: 30 minutes from prayer time (not from push send time — see Gray Area 2 below)

### Late Posting
- Users CAN post after the window closes — encouraged, not punished
- Late posts show 🕰 badge on feed card — funny, not demeaning
- No on-time badge (no ⭐ for on-time — only 🕰 for late)
- No shame language anywhere

### Camera
- Dual camera kept (front + back simultaneously) — already partially built
- Single capture (no retake tracking, no retake count display)

### Distribution
- 1 moment posts to ALL circles the user is in (not just one)
- One photo upload to storage (shared URL), one `circle_moments` row per circle
- Disclaimer on preview screen: "This will be shared to all your circles (N)" — show dynamic count

### What NOT to build in this phase
- Retake counter / tracking
- Late post shame ("X hours late")
- Public/cross-circle feed moments
- BeReal-style "see others only after posting" gate (Reciprocity Gate from Phase 8 handles this)
- Comment preview on moment card (deferred to post-MVP)

## Decisions Made in This Session

### 1. Notification Copy
Use warm Islamic tone. Example:
> "Time to capture this moment. Your circle is waiting. ✨"
> *(APNs title: circle name)*

Not punchy/short like a tweet — warm and inviting. The notification should feel intentional and Islamic in spirit, not urgent or gamified.

### 2. Prayer Selection Architecture
**Decision: Option B — global daily random via `daily_moments` table.**

- `daily_moments` table is the source of truth for which prayer fires today (populated by admin/cron)
- Edge function `send-moment-window-notifications` must be updated to read `daily_moments` for today's prayer (instead of `circles.prayer_time`)
- For each user, calculate the Aladhan time for that prayer using their lat/lng/timezone
- Fire notification when current time hits that prayer time (±2min window)
- `DailyMomentService` on the app side already reads `daily_moments` — keep as-is

The `circles.prayer_time` column is no longer the trigger source. The edge function needs to be rewritten to:
1. Query `daily_moments` for today's date to get `prayer_name`
2. For each circle member with location data, calculate that prayer's time
3. Fire notification at that prayer time

### 3. Multi-Circle Posting
**3a — Disclaimer wording:** Show dynamic count: "This will be shared to all your circles (3)"
- Use the user's actual circle count fetched from `circle_members`

**3b — Error handling on partial failure:**
- If upload succeeds but some circle inserts fail: show partial error ("Posted to X of Y circles"), offer retry
- Keep it simple — retry the failed inserts only (not re-upload)
- Don't treat the whole post as failed if at least one circle succeeded

**Implementation:** `MomentService.postMoment` needs to accept `circleIds: [UUID]` (all user's circles), upload once, loop inserts.

### 4. Feed Card Design (MomentFeedCard)

**4a — Card size:** Full-width BeReal style.
- Remove horizontal padding on the photo — fill screen width
- Tall 3:4 ratio (full width × 4/3 height)
- Front camera composited as small PiP overlay in corner (already composited before upload — the layout just needs to be full-width)
- No horizontal inset/padding on the photo itself

**4b — Badges:** Late badge only.
- 🕰 badge top-right corner for late posts (`isOnTime == false`)
- No ⭐ badge for on-time (remove existing ⭐ from `MomentCardView` too)

**4c — Own moment pinned to top:** After posting, user's own moment card stays pinned at the top of the circle feed — permanently (not just until scroll). Feed ordering: own moment first, then rest chronological.

## Known Blocker: RLS on circle_moments INSERT

**Error:** `"new row violates row-level security policy"` when tapping Post Moment.

**Root cause (likely):** `circle_moments` is missing an INSERT policy, or the policy doesn't match what the app sends. Also possible: storage bucket `circle-moments` has no INSERT policy for authenticated users.

**Fix (planner to audit and write migration):**
1. Audit `circle_moments` RLS policies in Supabase Dashboard
2. Ensure INSERT policy: `WITH CHECK (auth.uid() = user_id)`
3. Ensure SELECT policy: circle members can read moments in their circles
4. Audit storage bucket `circle-moments` — authenticated users must be able to upload to their own path
5. Multi-circle INSERT (one row per circle) must all pass the same RLS check

This is the **first task** in the phase — unblocks everything else.
